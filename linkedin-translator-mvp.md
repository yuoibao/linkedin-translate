## 1. 产品概述

### 1.1 产品定义

**一句话描述**：把"人话"和"LinkedIn腔"互相翻译的双向工具。

**完整描述**：一款面向求职者和LinkedIn运营者的双向翻译工具。用户可以输入大白话获得专业LinkedIn表述，也可以输入LinkedIn废话获得接地气的翻译。支持多种语气风格选择，适合金融、科技、咨询、创业等不同场景。

### 1.2 背景与动机

| 维度 | 描述 |
|------|------|
| **市场痛点** | LinkedIn已经成为职场社交标配，但很多人不知道如何写出专业的自我描述；另一方面，大量LinkedIn内容充斥着空洞的"废话"，阅读者苦不堪言 |
| **用户需求** | 求职者需要一个"翻译官"帮他们把平凡经历包装得亮眼；运营者需要快速理解或创作LinkedIn内容 |
| **产品机会** | 目前市面上缺乏简单好用的LinkedIn翻译工具，且缺乏中文语境的支持 |

### 1.3 目标愿景

> **"让每个人都能写出地道的LinkedIn文案，让阅读LinkedIn不再费脑。"**

## 2. 用户分析

### 2.1 目标用户画像

#### 用户群体A：求职者（Primary）

| 属性 | 描述 |
|------|------|
| **人口特征** | 22-35岁，大学毕业生至职场中期 |
| **使用场景** | 写简历、准备LinkedIn个人简介、更新工作经历 |
| **核心诉求** | 把平淡的工作内容描述得专业、亮眼、有说服力 |
| **痛点** | 不知道HR/招聘者想看什么，不擅长自我包装 |
| **使用频率** | 低频（求职季集中使用） |

#### 用户群体B：LinkedIn运营者（Primary）

| 属性 | 描述 |
|------|------|
| **人口特征** | 25-40岁，市场/运营/销售/创业者 |
| **使用场景** | 写帖子、想标题、理解竞品动态 |
| **核心诉求** | 快速产出或消化LinkedIn内容 |
| **痛点** | 创作耗时，阅读他人内容也费时 |
| **使用频率** | 中高频（定期使用） |

#### 用户群体C：猎头/HR（Secondary）

| 属性 | 描述 |
|------|------|
| **使用场景** | 快速理解候选人简历描述，或帮候选人优化文案 |
| **核心诉求** | 效率优先，理解内容是第一要务 |
| **使用频率** | 低频 |

### 2.2 用户故事（User Stories）

```
作为一名【求职者】
我想要【输入：我每天开会很多】
以便【获得专业的LinkedIn表述，提升简历吸引力】

---

作为一名【LinkedIn运营者】

以便【快速知道对方实际想说什么，节省阅读时间】
```

### 3.2 核心功能详解

#### F1：正向翻译（Human → LinkedIn）

| 字段 | 描述 |
|------|------|
| **功能ID** | F1 |
| **功能名称** | 正向翻译 |
| **功能描述** | 将用户输入的日常语言翻译成专业、地道的LinkedIn风格表述 |
| **触发方式** | 用户输入文本 + 点击"翻译"按钮 |
| **输入** | 文本（10-500字），语气风格（默认通用） |
| **输出** | 翻译后的LinkedIn风格文本 |
| **限制** | 单次最多500字，超出提示缩短 |

**正向翻译示例**：

| 输入（大白话） | 输出（LinkedIn腔） |
|----------------|-------------------|
| 我每天开会很多 | "在高节奏的工作环境中，主导并协调跨部门每日站会，确保信息同步与高效协同" |
| 我帮客户解决问题 | "为客户提供专业的解决方案，通过深入沟通精准把握需求并交付超出预期的价值" |
| 我改了PPT | "主导演示材料的视觉升级与内容优化，提升信息传递效率300%+" |

#### F2：反向翻译（LinkedIn → Human）

| 字段 | 描述 |
|------|------|
| **功能ID** | F2 |
| **功能名称** | 反向翻译 |
| **功能描述** | 将LinkedIn废话翻译成直白易懂的大白话，带轻微吐槽 |
| **触发方式** | 用户输入文本 + 点击"翻译"按钮 |
| **输入** | 文本（10-500字） |
| **输出** | 接地气的大白话翻译 + 轻微吐槽 |

**反向翻译示例**：

| 输入（LinkedIn腔） | 输出（人话+吐槽） |
|--------------------|------------------|
| "在快节奏的动态环境中发挥领导力" | "其实就是：有时候需要催催同事别摸鱼" |
| "颠覆性创新，引领行业变革" | "搞了点新东西，希望别亏本" |
| "打造闭环生态系统，实现赋能共赢" | "把事情做完了，希望有用吧" |

#### F3：语气风格选择

| 字段 | 描述 |
|------|------|
| **功能ID** | F3 |
| **功能名称** | 语气风格选择 |
| **功能描述** | 根据不同行业/场景选择不同的翻译语气 |
| **可选风格** | 通用（默认）、创业圈、咨询腔 |
| **交互方式** | 下拉选择，默认通用 |

**风格差异示例**（输入："我把东西卖出去了"）：

| 风格 | 输出 |
|------|------|
| 通用 | "达成销售目标，实现个人价值与公司收益的双赢" |
| 创业圈 | "在0到1的突破中，成功验证商业模式，实现营收破局" |
| 咨询腔 | "为客户提供精准的价值匹配方案，实现商业转化与持续复购" |

#### F4：模式切换

| 字段 | 描述 |
|------|------|
| **功能ID** | F4 |
| **功能名称** | 正/反向模式切换 |
| **交互方式** | Tab切换或滑块 |
| **切换行为** | 切换时清空输入框和输出框 |

#### F5：一键复制

| 字段 | 描述 |
|------|------|
| **功能ID** | F5 |
| **功能名称** | 一键复制 |
| **交互方式** | 点击复制按钮 |
| **反馈** | 显示"已复制 ✓"提示，2秒后消失 |
| **限制** | 无 |

#### F6：清空重置

| 字段 | 描述 |
|------|------|
| **功能ID** | F6 |
| **功能名称** | 清空重置 |
| **交互方式** | 点击"清空"按钮或快捷键 |
| **行为** | 清空输入框和输出框 |

### 3.3 辅助功能

| 功能 | 描述 | 优先级 |
|------|------|--------|
| 加载动画 | 翻译过程中显示加载状态 | P0 |
| 错误提示 | API失败时友好提示用户 | P0 |
| 空输入校验 | 未输入内容时禁用翻译按钮 | P0 |
| 移动端适配 | 手机/平板正常使用 | P0 |
| 打字统计 | 显示输入字符数 | P1 |

### 3.4 边界情况处理

| 场景 | 处理方式 |
|------|----------|
| 输入为空 | 禁用翻译按钮，提示"请输入内容" |
| 输入过短（<10字） | 提示"内容太短，请补充更多细节" |
| 输入过长（>500字） | 提示"内容过长，请控制在500字以内" |
| API超时（>10秒） | 显示"翻译超时，请重试" |
| API返回错误 | 显示"服务繁忙，请稍后重试" |
| 网络断开 | 显示"网络连接失败，请检查网络" |

### 5.3 API 设计

#### 翻译接口

```
POST /api/translate
Content-Type: application/json

请求体：
{
  "text": "我每天开会很多",
  "mode": "to_linkedin",        // "to_linkedin" | "to_human"
  "style": "general"             // "general" | "startup" | "consulting"
}

响应：
{
  "success": true,
  "result": "在高节奏的工作环境中，主导并协调跨部门每日站会..."
}
```

#### 错误响应

```json
{
  "success": false,
  "error": {
    "code": "TEXT_TOO_LONG",
    "message": "输入内容超过500字限制"
  }
}
```

### 5.4 错误码定义

| 错误码 | 含义 | 用户提示 |
|--------|------|----------|
| `TEXT_EMPTY` | 输入为空 | "请输入要翻译的内容" |
| `TEXT_TOO_SHORT` | 输入过短 | "内容太短，请补充更多细节" |
| `TEXT_TOO_LONG` | 输入过长 | "内容过长，请控制在500字以内" |
| `INVALID_MODE` | 模式参数错误 | "翻译模式无效，请刷新页面重试" |
| `API_ERROR` | AI服务错误 | "服务繁忙，请稍后重试" |
| `API_TIMEOUT` | AI服务超时 | "翻译超时，请重试" |
| `NETWORK_ERROR` | 网络错误 | "网络连接失败，请检查网络" |

### 5.6 AI Prompt 完整版

#### 正向翻译 Prompt（通用风格）

```
你是一个专业的LinkedIn内容撰写专家。请把用户输入的大白话翻译成专业、地道、LinkedIn风格的英文表述。

要求：
1. 保持真实，不要过度夸张
2. 使用LinkedIn常见表达（如：spearheaded, drove, enabled, delivered, leveraged等）
3. 适当量化成果（如果用户提到具体数字要保留）
4. 每句话突出一个重点，不要废话
5. 总字数控制在输入的1.5-2倍
6. 只输出翻译结果，不要解释

用户输入：{user_input}
```

#### 正向翻译 Prompt（创业圈风格）

```
你是一个活跃在创业圈的创始人，擅长用充满激情和颠覆性的语言撰写LinkedIn内容。

要求：
1. 多用"颠覆"、"创新"、"突破"、"赛道"、"破局"等词汇
2. 强调growth（增长）、impact（影响力）、scale（规模化）
3. 语气自信、有魄力
4. 保持真实，不要虚假夸大
5. 只输出翻译结果，不要解释

用户输入：{user_input}
```

#### 正向翻译 Prompt（咨询腔风格）

```
你是一个顶级咨询公司的顾问，擅长用咨询黑话撰写LinkedIn内容。

要求：
1. 多用"赋能"、"抓手"、"闭环"、"生态"、"协同"、"价值最大化"等词汇
2. 强调方法论、专业性、系统性
3. 善用"通过xxx实现yyy"的句式
4. 保持专业但不空洞
5. 只输出翻译结果，不要解释

用户输入：{user_input}
```

#### 反向翻译 Prompt

```
你是一个犀利的职场观察者，擅长把LinkedIn上的空洞废话翻译成直白的大白话。

要求：
1. 一针见血指出对方实际想表达的核心意思
2. 可以带轻微吐槽，但要幽默有度，不伤人
3. 用简洁直白的语言，不要废话
4. 格式：先用大白话翻译，然后换行写吐槽
5. 只输出翻译结果，不要额外说明

示例：
输入："Passionate about driving innovative solutions in a fast-paced environment"
输出：
大白话：工作挺忙的，有时候得催人干活
吐槽：所以你到底推动了啥创新？具体点会死吗

用户输入：{user_input}
```

## 8. 风险评估

### 8.1 风险矩阵

| 风险 | 概率 | 影响 | 等级 | 应对措施 |
|------|------|------|------|----------|

| 移动端体验差 | 低 | 高 | 🟡 中 | 重点测试移动端适配 |
| 竞品出现 | 中 | 中 | 🟡 中 | 快速迭代，保持差异化 |
| 域名被占用 | 低 | 高 | 🟢 低 | 提前注册备用域名 |

| 场景 | 应急措施 |
|------|----------|
| AI服务不可用 | 显示友好提示，提供"稍后重试"按钮 |
| 被刷API | 临时关闭服务，排查后添加人机验证 |
| 网站被墙 | 准备备选域名，通知用户 |
